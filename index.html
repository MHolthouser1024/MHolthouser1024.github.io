    Difference Between Switch and if/else

    When To Use A Switch Statement

    Similar to if statements, switch statements also compare variables with a value in Java.
    A switch statement starts with the keyword switch and then some parenthese.
    Within the parenthese is a single variable, and then followed up by a open curley brace.

    Inside the body of the switch statement are several case statements that begin with the keyword
    case and then a value that the variable up in the parentheses might equal. After all of
    the code is the keyword break. The break marks the end of the case.

    When a switch statement runs, the computer figures out the current value of the variable
    inside the parentheses. Then it looks through the list of cases, one at a time, looking for a match.
    When it finds a match it moves from the left side where the cases are to the right side and starts
    running code until it is stopped by a break.

    I none of the cases match and there is a default case, which is optional, then the code in the default
    case will be run instead.

    What Is An if/else Statement

    An if statement starts with the keyword if, followed by a condition in parenthese.
    The condition must be a boolean expression that evaluates to either true or false. Underneath that
    starts a block of code surrounded by curly braces, and everything inside the curley braces is called
    the body of the if statement.

    When the condition of the if statement is true, all the code in the body of the if statement is executed.
    When the condition of the if statement is false, all the code in the body is skipped. The amount of code is
    unlimited within the body of the if statement.

    So...what's the difference?

    So basically, a switch statement works much faster than the equivalent if/else ladder. Switch statements
    are easier to read than if/else statements, and are typically more manageable.

    You will want to use a switch statement over a if/else statement when there are large number of compares
    for a condition in your program, or when there are more complex comparisons. However, when there are just
    a few compares, the if/else statements are faster and easier to write.
